# Remote Radar MVP Design


## Goal

实现基础功能：

* 抓取电鸭远程工作 -> LLM 归一化/打标/处理 -> 信息订阅。
* 实现脚本触发全量抓取工具。



## Features

* **抓取模块**

> 负责从各个招聘渠道抓取信息，抓取后直接入库存储原始抓取信息。首次全量抓取，后续增量抓取。

本期支持从电鸭平台抓取数据，简单处理后直接入表存储（存储于电鸭数据原始表）



* **特征处理模块**

> 根据原始的抓取数据，根据规则 + LLM 对数据做处理，并抽象成 Job Model 存储到数据库里面
> Raw → Clean → Normalize → LLM Tag → Final Model

本期支持：

* 根据抓取模块的内容 Pipeline
  * 基础的规则清洗，如是否命中关键字（关键字支持配置）
  * 前后内容 Trim
* 调用 Deepseek 做清洗（Prompt 在配置文件里配置）
  * 判断是否为一个远程招聘信息，如果 LLM 判断不是，那么舍弃内容
  * 生成 Tag 信息（tag 候选在配置文件中配置）
  * 生成结构化信息：
    * 全职 / 兼职
    * 薪水区间（美元）：小于 \$1000 / \$1000 \~ \$5000 / \$5000 \~ \$10000 / \$10000 以上
    * 岗位内别（支持配置）：后端开发工程师 / 前端开发工程师 / 全栈开发工程师 / 其他
    * 语言要求：英语 / 日语 / 无
    * 推荐星级：5 （最高 5 分，请从专业的 HR 角度考虑职位性价比）
    * 要求技能 Tag：根据 JD 生成技能要求 Tag



* **展示模块**

本期支持：提供前端展示页面，供用户筛选、查阅职位信息



* **推送模块**

本期支持：

* 提供筛选推送订阅能力，用户可以根据需求在页面上订阅指定 Tag 职位更新信息
* 支持推送方式选择：Log 模式（for debug），邮箱模式（用户在页面填入邮箱）

## Architecture

Scrape → Clean → Tag → Store → View → Notify



## Non-goals

* 不做注册登录、不做多用户
* 不做复杂前端功能
* 不做 LLM 分析、Chat 功能
